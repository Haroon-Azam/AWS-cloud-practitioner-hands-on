## Objective
To store objects in Amazon S3 and configure controlled public access using bucket policies.

---

## AWS Services Used
- Amazon S3
- AWS Policy Generator

---

## Tasks Performed
- Created an Amazon S3 bucket
- Uploaded an image object to the bucket
- Verified default private access behavior
- Disabled Block Public Access settings
- Generated and applied a bucket policy to allow public read access
- Accessed the object using a public URL

---

## Why S3 is Important
Amazon S3 provides scalable, durable, and cost-effective object storage. It is commonly used for static content hosting, backups, and media storage.

---

## Outcome
- Successfully configured public access for an S3 object
- Verified access via public URL
- Understood S3 security and permission controls

---

## Notes
- Public access was enabled only for demonstration purposes
- Bucket policies were used instead of object-level ACLs
- No sensitive data is stored
